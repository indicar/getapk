# File Upload/Download and Android Request Handler

Простой клиент и сервер для загрузки, скачивания файлов и обработки запросов от Android клиента.

## Описание

Этот проект включает в себя:
- Сервер (app.py) с возможностью загрузки и скачивания файлов
- Клиент (client.py) для взаимодействия с сервером
- Поддержку обработки запросов от Android клиента (текст и/или изображения)
- Управление URL сервера через API

## Установка

1. Клонируйте репозиторий
2. Установите зависимости: `pip install -r requirements.txt`
3. Создайте файл `.env` с настройками (см. ниже)

## Настройка

Создайте файл `.env` в корне проекта со следующими переменными:

```
SERVER_URL=http://your-server-url.com
API_USERNAME=your_username
API_PASSWORD=your_password
```

## Запуск сервера

```bash
python app.py
```

Сервер будет доступен по адресу, указанному в переменной PORT (по умолчанию 5000), например: http://localhost:5000

## Использование клиента

Запустите клиент:

```bash
python client.py
```

Для тестирования различных функций раскомментируйте соответствующие строки в блоке `if __name__ == "__main__":`.

## Функции клиента

- `upload_file(file_path)` - загрузка файла на сервер
- `download_file(output_path=None)` - скачивание последнего загруженного файла с сервера
- `check_health()` - проверка работоспособности сервера
- `check_download_capability()` - проверка возможности скачивания файлов с сервера
- `set_server_url(new_url)` - установка URL сервера на сервере (через endpoint /set_url)
- `get_server_url()` - получение текущего URL сервера с сервера (через endpoint /get_url)
- `send_request(text=None, image_path=None)` - отправка запроса от Android клиента (через endpoint /send_request)
- `get_request_status()` - получение статуса последнего запроса (через endpoint /request_status)
- `get_last_request()` - получение последнего запроса и пометка его как прочитанного (через endpoint /get_last_request)

При скачивании файла, если не указан путь для сохранения, имя файла будет определено автоматически:
- Из заголовка Content-Disposition, если сервер его предоставляет
- Или на основе Content-Type с именем "file" и соответствующим расширением

## Серверные endpoint'ы

Сервер предоставляет следующие endpoint'ы:

- `/upload` - загрузка файла на сервер (требует аутентификацию)
- `/download` - скачивание последнего загруженного файла (требует аутентификацию)
- `/health` - проверка работоспособности сервера
- `/set_url` - установка URL сервера (требует аутентификацию)
- `/get_url` - получение текущего URL сервера (требует аутентификацию)
- `/send_request` - отправка запроса от Android клиента (текст и/или изображение) (требует аутентификацию)
- `/request_status` - получение статуса последнего запроса (требует аутентификацию)
- `/get_last_request` - получение последнего запроса и пометка его как прочитанного (требует аутентификацию)
- `/docs/` - документация API (Swagger UI)

Все endpoint'ы, кроме `/health` и `/docs/`, требуют базовую HTTP-аутентификацию с использованием API_USERNAME и API_PASSWORD.

## Android клиент интеграция

Сервер предоставляет специальные endpoint'ы для интеграции с Android клиентом:

1. `/send_request` - принимает текстовые и графические запросы от Android клиента
2. `/request_status` - позволяет проверить наличие новых запросов
3. `/get_last_request` - позволяет получить последний запрос и отметить его как прочитанный

Эти endpoint'ы позволяют системе на стороне сервера обрабатывать запросы от Android клиента, содержащие как текстовую, так и графическую информацию, и направлять их в LLM для дальнейшей обработки.

## Документация API

Полная документация API доступна по адресу: `http://your-server-address/docs/` (Swagger UI)
